页面架构方案

从单页应用到多页面产品的路径

A

Next.js App Router 多页面（推荐）

利用 Next.js App Router 的嵌套布局，共享侧边栏 layout，每个功能独立路由。

app/
├── (main)/ ← 共享侧边栏的 layout group
│ ├── layout.tsx ← 侧边栏 + 主内容区
│ ├── page.tsx ← 首页（会议列表）
│ ├── meeting/[id]/
│ │ └── page.tsx ← 文档编辑页
│ ├── chat/
│ │ └── page.tsx ← AI 对话页
│ └── settings/
│ └── page.tsx ← 设置页
└── layout.tsx ← 根 layout
✓ URL 可分享 ✓ 浏览器前进/后退 ✓ 代码分割 ✓ Next.js 原生模式
B

客户端状态路由（SPA）

保持单页应用，用 Zustand 状态切换"视图"。侧边栏点击只改变 state，不跳转页面。

app/
├── page.tsx ← 唯一页面
└── layout.tsx

components/
├── AppShell.tsx ← 侧边栏 + 视图容器
├── views/
│ ├── HomeView.tsx ← 会议列表
│ ├── MeetingView.tsx ← 文档编辑
│ ├── ChatView.tsx ← AI 对话
│ └── SettingsView.tsx ← 设置
✓ 改动最小 ✓ 切换无闪烁 ✗ URL 不可分享 ✗ 无浏览器导航

方案对比

方案 A 优势

  • 每个页面有独立 URL（/meeting/abc123）
  • 浏览器前进/后退自然工作
  • Next.js 自动代码分割，首屏更快
  • 符合 Next.js App Router 最佳实践
  • 未来可加 SSR/SSG 优化

方案 B 优势

  • 改动量最小，复用现有代码
  • 页面切换零延迟
  • 状态管理更简单（Zustand 直接控制）
  • 不需要处理路由间状态传递
