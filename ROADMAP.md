# AI Notepad vs Granola 功能对标与演进路线

更新时间：2026-02-27

---

## 一、功能对标矩阵

> ✅ 已实现 | 🟡 部分实现 | ❌ 未实现 | ➖ 刻意不做

| # | 功能模块 | Granola | AI Notepad | 差距说明 |
|---|---------|---------|------------|---------|
| 1 | **音频采集（本地麦克风+系统音频）** | ✅ 桌面端本地采集，无机器人入会 | ✅ 双通道采集（麦克风+系统音频），浏览器端实现 | **持平**。Granola 为原生客户端；我们为 Web 端，依赖浏览器权限 |
| 2 | **语音转写（ASR）** | ✅ 桌面端实时转写；iOS 会后转写。使用 Deepgram / AssemblyAI | ✅ 双模式：Browser Web Speech（Demo）+ 阿里云 ASR（生产）实时 WebSocket 流式转写 | **持平**。我们已完成阿里云 ASR 全链路接入（16kHz PCM、双通道），与 Granola 桌面端能力对等 |
| 3 | **手动触发/启动** | ✅ 需手动点击开始，不会自动进会 | ✅ 需用户手动点击"开始录音"，支持暂停/恢复 | **持平** |
| 4 | **转写分段（Chunking）** | ✅ UI 以气泡/chunk 呈现，可复制/删除单个 chunk | 🟡 以 `TranscriptSegment` 分段存储，UI 展示气泡但**不支持**逐段复制/删除操作 | **差距**：缺少单段操作（复制/删除） |
| 5 | **说话人分离（Diarization）** | 🟡 iOS 支持最多 10 人；桌面端不支持 | 🟡 阿里云 ASR 支持说话人分离；支持说话人重命名（SpeakerManager） | **持平偏优**。我们桌面端已有分离能力（通过阿里云），Granola 桌面端反而不支持 |
| 6 | **时间戳** | 🟡 API 返回 start_time/end_time；UI 展示未确认 | ✅ 每段存储 startTime/endTime，UI 目前未显著展示 | **持平**。数据已有，UI 展示可增强 |
| 7 | **转写编辑** | ❌ 官方明确"不支持编辑转写" | ❌ 同样不支持编辑转写文本 | **持平**（双方均不支持） |
| 8 | **转写内搜索** | ✅ 转写面板内关键词搜索 | ❌ 转写面板无搜索功能 | **差距**：需新增转写搜索 |
| 9 | **导出格式** | 🟡 复制/邮件分享；CSV 导出（不含全转写，24h 时效） | ✅ Markdown 导出 + DOCX 导出 + 剪贴板复制 | **我方领先**。我们支持结构化 DOCX，Granola 仅 CSV 且限制多 |
| 10 | **实时/离线** | 🟡 桌面端实时转写；笔记本地缓存可离线编辑 | 🟡 实时转写需联网（阿里云 ASR）；笔记本地 SQLite 可离线读写 | **持平** |
| 11 | **会议结束自动检测** | ✅ 检测通话结束、15 分钟无音频、设备休眠 | ❌ 仅支持手动停止 | **差距**：缺少自动停止机制 |
| 12 | **AI 增强笔记（总结/结构化）** | ✅ 会后自动生成，输入为转写+手写笔记+日历信息 | ✅ 会后一键生成，输入为转写+手写笔记+会议标题。支持 5 种会议类型 + 4 种输出风格 | **持平偏优**。我们可定制化程度更高（类型+风格+行动项开关），但缺少日历信息融合 |
| 13 | **模板系统** | ✅ 桌面端支持模板；无法设默认模板 | ✅ 5 个系统模板 + 用户自定义模板，`/` 命令调用，Prisma 持久化 | **持平**。功能对等 |
| 14 | **Chat（单会议问答）** | ✅ 基于转写+笔记+增强笔记的问答 | ✅ 流式回复，融合转写+手写笔记+增强笔记+聊天历史 | **持平** |
| 15 | **Chat（跨会议/全局）** | ✅ 支持文件夹/全局作用域；受"25 个最近会议"限制 | ✅ 关键词+向量混合检索；按时间/标题过滤；来源引用 | **持平**。架构相似 |
| 16 | **Chat 模型选择** | ✅ Auto 模式（GPT-5.1 / Claude 4 Sonnet）；可手动切换 | 🟡 Gemini 主通道 + MiniMax 回退；**不支持用户手动切换** | **差距**：缺少前端模型选择器 |
| 17 | **语音输入（Dictation）** | ✅ Chat 栏麦克风口述提问 | ❌ Chat 输入仅支持文字 | **差距**：缺少语音输入 |
| 18 | **多语言转写** | ✅ 桌面端可切换多语言 | 🟡 阿里云 ASR 配置为中英混合，**不支持动态切换** | **差距**：缺少语言切换 UI |
| 19 | **内部术语/热词增强** | ✅ 设置中提交公司术语提升准确率 | ❌ 未实现 | **差距**：缺少热词管理 |
| 20 | **分享与协作** | ✅ 分享链接+访问级别；共享文件夹 | 🟡 预留飞书/企业微信 Webhook（**未完整实现**） | **差距**：分享能力未落地 |
| 21 | **文件夹/工作区** | ✅ 工作区隔离+文件夹共享 | ❌ 仅平铺会议列表 | **差距**：缺少组织结构 |
| 22 | **集成（Notion/Slack/HubSpot 等）** | ✅ 多平台集成 | ❌ 仅预留 Webhook | **差距**：无外部集成 |
| 23 | **MCP 连接器** | ✅ 对外暴露会议数据给 AI 工具 | ❌ 未实现 | **差距** |
| 24 | **API/SDK** | ✅ 企业 REST API（只读） | ❌ 无对外 API | **差距**（优先级低） |
| 25 | **音频留存** | ❌ 不存储音频（隐私设计） | ❌ 同样不存储音频 | **持平**（双方设计一致） |
| 26 | **安全与合规** | 🟡 SOC 2 Type II、GDPR、非 HIPAA | ➖ 本地 SQLite，数据不出设备 | **我方优势**。纯本地存储天然满足数据隐私 |

## 二、差距总结

### 我方优势（vs Granola）
1. **导出能力更强** — 支持结构化 DOCX，Granola 仅 CSV
2. **桌面端说话人分离** — 阿里云 ASR 已支持，Granola 桌面端不支持
3. **AI 定制化更灵活** — 会议类型 × 输出风格 × 行动项开关的组合
4. **数据隐私** — 纯本地 SQLite，零云端数据泄露风险
5. **开源可控** — 可自由扩展和二次开发

### 我方差距（需弥补）
| 优先级 | 差距项 | 影响 |
|--------|-------|------|
| **P0** | 转写面板搜索 | 基础体验缺失 |
| **P0** | 会议自动结束检测 | 用户体验痛点 |
| **P0** | 转写单段操作（复制/删除） | 基础交互缺失 |
| **P1** | 文件夹/分组管理 | 会议多了无法组织 |
| **P1** | 飞书/企微分享落地 | Webhook 已预留但未对接 |
| **P1** | 语音输入（Chat Dictation） | 提效功能 |
| **P1** | 热词/术语增强 | 专业场景转写准确率 |
| **P2** | 前端模型选择器 | 高级用户需求 |
| **P2** | 多语言切换 | 国际化场景 |
| **P2** | MCP 连接器 | 生态扩展 |
| **P3** | 外部集成（Notion/Slack 等） | 工作流闭环 |
| **P3** | 对外 API | 企业场景 |

## 三、演进路线（Phase Plan）

### Phase 1：基础体验补全（1-2 周）

> 目标：补齐核心使用流程中的短板，让产品从"能用"到"好用"。

#### 1.1 转写面板搜索（P0）
- 在 `TranscriptPanel` 顶部增加搜索栏
- 支持关键词高亮定位
- 支持上/下箭头在匹配项间跳转
- 无匹配时显示"未找到"提示

#### 1.2 转写单段操作（P0）
- 每段 Hover 时显示操作菜单（复制文本 / 删除该段）
- 复制：复制"[说话人] 内容"到剪贴板
- 删除：二次确认后从 store + DB 移除该段

#### 1.3 会议自动结束检测（P0）
- 检测逻辑：连续 N 分钟（默认 10 分钟）无新转写段产生 → 自动弹出提示"检测到会议可能已结束，是否停止录音？"
- 用户可选择"停止"或"继续录音"
- 系统音频断开事件（如通话软件退出）也触发提示
- 可在设置中调整超时时长或关闭此功能

#### 1.4 时间戳 UI 展示
- 在转写段左侧或说话人标签旁显示相对时间戳（如 `02:34`）
- Hover 时显示绝对时间

---

### Phase 2：内容组织与管理（2-3 周）

> 目标：解决会议数量增长后的管理问题，并完善分享能力。

#### 2.1 文件夹/分组管理（P1）
- 数据库增加 `Folder` 模型（id, name, color, sortOrder, createdAt）
- Meeting 增加可选 `folderId` 外键
- 侧边栏支持：创建文件夹 → 拖拽会议到文件夹 → 展开/折叠文件夹
- "未分组"会议单独展示
- 跨会议 Chat 支持按文件夹作用域过滤

#### 2.2 飞书/企业微信分享落地（P1）
- 完善 `/api/share/webhook` 的飞书富文本卡片格式
- 完善企业微信 Markdown 消息格式
- 前端增加分享配置入口（Webhook URL 管理）
- 分享预览：发送前可预览消息格式
- 分享状态反馈：成功/失败通知

#### 2.3 会议搜索与过滤
- 侧边栏增加全局搜索框
- 搜索范围：会议标题 + 增强笔记内容
- 按日期范围过滤
- 按文件夹过滤

---

### Phase 3：智能增强（3-4 周）

> 目标：提升 AI 能力上限，让"智能"成为差异化优势。

#### 3.1 热词/术语增强（P1）
- 新增 `Glossary` 模型（id, term, pronunciation, category）
- 设置页面支持批量添加/导入术语
- 阿里云 ASR 热词表动态注入（通过 vocabulary_id 参数）
- LLM Prompt 中注入术语表作为上下文，提升总结准确度

#### 3.2 语音输入 — Chat Dictation（P1）
- Chat 输入框旁增加麦克风按钮
- 使用 Browser Web Speech API 进行语音转文字（与主录音独立）
- 识别完成后自动填入输入框
- 视觉反馈：录音时输入框发光/脉冲动效

#### 3.3 前端 LLM 模型选择器（P2）
- Chat 面板或设置面板增加模型切换 UI
- 选项：Auto（默认）/ Gemini / MiniMax / 自定义
- 选择存入 Zustand store，请求时携带 `model` 参数
- 后端 `llm-provider.ts` 支持按请求指定 provider

#### 3.4 自动生成会议标题（新增能力）
- 录音结束后，如标题仍为默认值（"新会议"），自动用转写前 500 字调用 LLM 生成简短标题
- 用户可编辑/覆盖

---

### Phase 4：生态与集成（4-6 周）

> 目标：从单机工具走向可连接的工作流节点。

#### 4.1 MCP 连接器（P2）
- 实现 MCP Server，暴露以下 Resources：
  - `meetings/list` — 会议列表
  - `meetings/{id}` — 单会议详情（含转写+笔记）
  - `meetings/search` — 按关键词/日期搜索
- 支持 OAuth 或 Token 鉴权
- 外部 AI 工具（如 Claude Desktop）可直接检索会议上下文

#### 4.2 多语言转写切换（P2）
- 前端增加语言选择器（中文 / 英文 / 日文 / 中英混合 等）
- 选择后动态调整阿里云 ASR 参数
- 笔记增强 Prompt 自动适配目标语言

#### 4.3 外部集成 — Notion（P3）
- OAuth 接入 Notion API
- "导出到 Notion"按钮：将增强笔记作为 Page 写入指定 Database
- 字段映射：标题 → Title，摘要 → Rich Text，行动项 → To-do

#### 4.4 外部集成 — 邮件发送（P3）
- 集成 Resend / Nodemailer
- 一键将会议纪要发送给参会人
- 支持自定义收件人列表 + 邮件模板

---

### Phase 5：进阶产品化（远期）

> 目标：从工具走向平台，支撑团队级使用。

#### 5.1 用户系统与多端同步
- 增加用户注册/登录（可选 OAuth）
- 数据库从 SQLite 迁移至 PostgreSQL
- 会议数据云端同步，支持多设备访问

#### 5.2 协作与权限
- 会议笔记可分享给团队成员
- 访问级别：仅查看 / 可编辑 / 管理员
- 评论/批注功能

#### 5.3 日历集成
- 对接 Google Calendar / Outlook
- 自动创建会议记录卡片
- 日历信息注入 AI 总结（参会人、议程等）

#### 5.4 移动端适配
- 响应式布局优化 or 独立 PWA
- 移动端精简模式：查看历史 + Chat 问答
- 移动端录音（类 Granola iOS 会后转写模式）

#### 5.5 对外 API（企业版）
- REST API：笔记查询、模板管理
- API Key 管理与限流
- Webhook 回调：会议结束时通知外部系统

## 五、技术风险与注意事项

| 风险点 | 说明 | 应对策略 |
|--------|------|---------|
| **ASR 中转架构** | 当前浏览器直连阿里云 WebSocket，Token 暴露风险 | Phase 2 考虑服务端 WebSocket 中转 |
| **向量检索质量** | 当前使用本地 384 维向量，语义理解有限 | Phase 3 可接入云端 Embedding（如 Gemini Embedding） |
| **SQLite 并发** | 单文件数据库在多标签页/多用户场景下可能锁冲突 | Phase 5 迁移 PostgreSQL 前，限制单实例运行 |
| **浏览器音频权限** | System Audio Capture 依赖 `getDisplayMedia`，兼容性有限 | 文档明确标注支持的浏览器版本 |
| **LLM 成本** | 跨会议 Chat 可能大量消耗 Token | 增加上下文截断策略 + Token 用量统计 |
